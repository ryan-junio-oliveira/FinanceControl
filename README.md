## About Laravel

Laravel is a web application framework with expressive, elegant syntax. We believe development must be an enjoyable and creative experience to be truly fulfilling. Laravel takes the pain out of development by easing common tasks used in many web projects, such as:

- [Simple, fast routing engine](https://laravel.com/docs/routing).
- [Powerful dependency injection container](https://laravel.com/docs/container).
- Multiple back-ends for [session](https://laravel.com/docs/session) and [cache](https://laravel.com/docs/cache) storage.
- Expressive, intuitive [database ORM](https://laravel.com/docs/eloquent).
- Database agnostic [schema migrations](https://laravel.com/docs/migrations).
- [Robust background job processing](https://laravel.com/docs/queues).
- [Real-time event broadcasting](https://laravel.com/docs/broadcasting).

Laravel is accessible, powerful, and provides tools required for large, robust applications.

## Custom Features

This project now includes a simple role/permission layer powered by [spatie/laravel-permission](https://github.com/spatie/laravel-permission).

- run `composer require spatie/laravel-permission` and publish the resources (`php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider" --tag="permissions-migrations"`) before migrating.
- migration for the permission tables is already added in `database/migrations/2026_02_23_000000_create_permission_tables.php`.
- seeds create three roles (`root`, `gestor`, `member`) and assign them when users register or are invited.
- the `root` user has access to system configuration; organizational owners (gestores) manage their own organisations.

Notifications/alerts are generated by a scheduled command:

```bash
php artisan alerts:check
```

It will send e‑mails and database notifications when budgets are exceeded, credit card invoices are near due, or the organization runs a negative balance. The command is scheduled daily at 01:00 via the scheduler.

### Módulo de administração do sistema (root)

A funcionalidade administrativa foi promovida a um módulo completo similar aos demais:

- **Estrutura em camadas** sob `app/Modules/Admin`:
  - `Domain` com entidades e contratos de repositório (Bank, Category).
  - `Infrastructure/Persistence/Eloquent` com implementações de repositório.
  - `Application/UseCases` servindo como casos de uso (ListBanks, ManageCategory, etc.) e contratos.
  - `Presentation` contendo controladores, rotas e views específicas.
- Rotas do módulo estão em `app/Modules/Admin/Presentation/Routes/web.php` e são carregadas automaticamente pelo `ModuleServiceProvider`.
  > Observação: os módulos originais `Bank` e `Category` ainda existem para suportar lógica de domínio e infraestrutura usada pelos demais módulos (como cartões de crédito e orçamentos), mas suas camadas de apresentação (controllers/rotas/requests) foram removidas em favor do módulo `Admin`.
O painel acessível pela role `root` permite:

- Gerenciar **Bancos** (CRUD global de nomes de bancos).
- Gerenciar **Categorias** de qualquer organização (com filtro e seleção de organização).

Atenção: o sistema também usa o canal `database` de notificações. Uma migração para criar a tabela `notifications` foi adicionada (ex: `2026_02_23_010000_create_notifications_table.php`). Rode `php artisan migrate` sempre que atualizar para garantir que a tabela exista.

Acesso via `/admin/banks` e `/admin/categories`; as rotas estão protegidas por `role:root`. Visite `/admin/settings` para a página de configurações inicial.
